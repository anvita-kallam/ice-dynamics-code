# Making the Archive VI SSA match icepack spin-up

This note explains every physics change we made so the **Archive variational inference (VI)** code speaks the same language as **icepack**, the forward model used in the spin-up notebooks.

It is written for someone new to glaciology. Math symbols are kept light; think of “forcing ice to slide downhill” first, equations second.

> **Math rendering:** Display equations use `$$...$$` and inline math uses `$...$` (GitHub, VS Code, Jupyter).
>
> **Where to look in code:** every altered block in Archive is marked with a `# CHANGED` comment in `Archive/models_torch.py` and `Archive/utilities_torch.py`.

---

## Mini glossary

| Term | Plain meaning |
|------|----------------|
| **Ice sheet / ice stream** | Thick ice that creeps and/or slides under its own weight |
| **SSA** | *Shallow Shelf / Stream Approximation* — a 2D model for fast horizontal flow (plug flow), used when ice is sliding a lot |
| **Icepack** | Python library that solves SSA with finite elements (Firedrake). Our spin-up ground truth comes from icepack |
| **VI + PINN** | Machine-learning setup that learns fields (like viscosity) so that (1) data fit and (2) physics residuals are small |
| **Viscosity** $\eta$ | How “stiff” the ice is — higher $\eta$ → harder to stretch/shear |
| **Fluidity** $A$ | Glen-flow parameter: how easily ice deforms for a given stress (icepack’s main rheology input) |
| **Basal friction** $C$ | How sticky the bed is (sliding resistance) |
| **Driving stress** | Pull from the ice-surface slope (gravity acting on thickness × slope) |
| **Basal drag** | Resistance at the bed that balances that pull when ice is sliding |
| **Membrane stress** | Horizontal stretching/shearing stresses inside the ice column (depth-averaged) |

---

## Why we changed the VI code (and not icepack)

1. Spin-up data is already generated by **icepack**.
2. VI training needs **autograd** inside PyTorch — icepack’s FEM solver is not that.
3. So we kept icepack as the truth generator and **aligned VI’s residual equations** with icepack’s physics family.

If VI and icepack use different units, basal drag, or rheology, VI can “succeed” at minimizing a *wrong* residual and infer the wrong viscosity.

---

## Snapshot: before vs after

| Piece | Original Archive VI SSA | After alignment |
|-------|-------------------------|-----------------|
| Units | SI-ish (Pa, m/s²) while data was often m/yr | icepack (m, yr, MPa) |
| Driving stress | $-\rho g H \nabla s$ with SI $g$ | $+\rho_I g H \nabla s$ with icepack-scaled densities/$g$ |
| Ice stiffness | Linear in learned $\eta$ only | Glen-style membrane; $\mu = \eta$ (default) or $\mu_{\mathrm{Glen}}(A,\dot\varepsilon)$ |
| Sliding | Learned fraction $\lambda$ times driving stress | Spin-up **plastic** basal law with fixed $C$ |
| Continuity $\nabla\cdot(H\mathbf{u})=0$ | Always in the loss | Optional (off by default; icepack diagnostic SSA does not enforce it) |
| Parameters $A$, $C$ | Not loaded from spin-up | Read from NPZ `cfg_json` |

## Change 1 — Units (meters, years, megapascals)

### Why

icepack measures velocity in **meters per year** and stress-like quantities in **megapascals (MPa)**. The original VI residual used SI constants ($g \approx 9.8\,\mathrm{m/s^2}$, viscosity in Pa·s). Mixing those with spin-up NPZ velocities (m/yr) makes residuals huge or meaningless.

**Analogy:** weighing yourself in kilograms but plugging pounds into a formula that expects kilograms.

### Before

```python
rho_ice = torch.tensor(917.0, ...)
gravity = torch.tensor(9.80665, ...)
tau_dx = -rho_ice * gravity * H * s_x
# eta_init ~ 1e6 (Pa · s style prior)
```

### After

```python
# CHANGED: icepack constants (m, yr, MPa)
icepack = icepack_ssa_constants(pars, torch_dtype, X.device)
rho_ice = icepack['rho_ice']      # density scaled like icepack
gravity = icepack['g']            # g scaled with year**2
tau_dx = rho_ice * gravity * H * s_x
```

Also in defaults / config:

```ini
# CHANGED: effective viscosity prior in MPa·yr
eta_init = 1.0          # was ~1e6
eta_min = 1.0e-3
eta_max = 1.0e6
```

And when loading data:

```python
# CHANGED: keep velocity in m/yr (do not convert to m/s)
u = data['ux']
v = data['uy']
```

### What stayed the same in spirit

Ice still “pulls” toward thicker/higher surface slopes; we only made the **numbers** agree with icepack.

## Change 2 — Driving-stress sign and form

### Why

In icepack’s variational `IceStream` gravity term, the relevant force density is proportional to

$$
\rho_I\, g\, H\, \nabla s.
$$

(The Euler–Lagrange sign is arranged so this balances membrane + friction.) The old VI residual used a minus in front of $\rho g H \nabla s$ in SI units. After unit fixes, we set

$$
\boldsymbol{\tau}_d = \rho_I\, g\, H\, \nabla s
$$

so the residual is

$$
\nabla\cdot(\text{membrane}) + \boldsymbol{\tau}_d - \boldsymbol{\tau}_b \approx 0.
$$

### Before

```python
tau_dx = -rho_ice * gravity * H * s_x
tau_dy = -rho_ice * gravity * H * s_y
```

### After

```python
# CHANGED: driving stress matches icepack IceStream.gravity
tau_dx = rho_ice * gravity * H * s_x
tau_dy = rho_ice * gravity * H * s_y
```

### Plain meaning

Surface slope creates the “downhill” pull; thickness $H$ scales how strong that pull is (more ice = more weight).

## Change 3 — Membrane / “stretching” stresses (Glen shape)

### Why

Ice does not stretch like a linear spring. Under Glen’s law ($n=3$), effective viscosity depends on how fast ice is straining:

$$
\mu = \frac{1}{2}\, A^{-1/n}\, \dot\varepsilon_e^{\,1/n - 1}.
$$

The original VI used the *MacAyeal linear* form with a **constant** $\eta$ plugged in:

$$
\boldsymbol{\sigma} \propto H\,\eta\,(\text{derivatives of } u, v).
$$

That form is fine as a *template* for the tensor, but treating $\eta$ as unrelated to icepack’s Glen effective viscosity mixed two constitutive stories.

### Before

```python
membrane_xx = 2.0 * H * eta * (2.0 * u_x + v_y)
membrane_xy = H * eta * (u_y + v_x)
membrane_yy = 2.0 * H * eta * (u_x + 2.0 * v_y)
# (always η from the GP; no Glen ε_e, no A)
```

### After

```python
# CHANGED: Glen ε_e and optional μ_Glen(A, ε)
eps_eff = glen_effective_strain_rate(u_x, u_y, v_x, v_y, eps_min)
mu_glen = glen_effective_viscosity(eps_eff, fluidity_A, glen_n)
use_inferred_eta = getattr(pars.prior, 'ssa_use_inferred_eta', True)

mu = eta if use_inferred_eta else mu_glen
membrane_xx = 2.0 * H * mu * (2.0 * u_x + v_y)
# ... same tensor structure as before ...
```

### Why keep $\mu = \eta$ by default?

Spin-up NPZs save **effective viscosity** as a diagnostic field. The VI *goal* is often to recover that field. So:

- **`ssa_use_inferred_eta = True` (default):** $\mu =$ learned $\eta$ → inverse problem for viscosity.
- **`ssa_use_inferred_eta = False`:** $\mu =$ forward Glen $\mu(A,\dot\varepsilon)$ with $A$ from config → check against icepack’s forward rheology.

New helpers:

```python
# CHANGED helpers in models_torch.py
glen_effective_strain_rate(...)
glen_effective_viscosity(...)
```

## Change 4 — Basal sliding law (the big one)

### Why

**Original VI:** ice slides with a fraction $1-\lambda$ of the *driving stress*, aligned with velocity. $\lambda$ was a learned map:

$$
\boldsymbol{\tau}_b = (1-\lambda)\,|\boldsymbol{\tau}_d|\,\frac{\mathbf{u}}{|\mathbf{u}|}.
$$

**icepack default Weertman:**

$$
\boldsymbol{\tau}_b \propto C\,|\mathbf{u}|^{1/m-1}\,\mathbf{u}.
$$

**Our spin-up notebooks:** a **regularized plastic** (hybrid) friction law using effective pressure $N$ and friction coefficient $C$ — *not* the $\lambda$-law and not bare Weertman.

VI was inferring sliding with a different story than the simulator that made the truth.

### Before

```python
tau_eff = sqrt(tau_dx**2 + tau_dy**2).clamp(min=tau_floor)
# ... λ sampled from a GP ...
basal_drag_x = (1.0 - lam) * tau_eff * u / speed
basal_drag_y = (1.0 - lam) * tau_eff * v / speed
rux = membrane_div_x + tau_dx - basal_drag_x
```

(plus nested loop over $\lambda$ quadrature samples)

### After

```python
# CHANGED: yield-like stress from effective pressure (spin-up notebooks)
water_depth = clamp(-(s - H), min=0)
tau_c = 0.5 * clamp(p_ice - p_water, min=0)

# CHANGED: plastic basal drag; C fixed (no λ in SSA residual)
basal_drag_x, basal_drag_y = spinup_plastic_basal_drag(
    u, v, speed, tau_c, friction_C, weertman_m, speed_eps
)
rux = membrane_div_x + tau_dx - basal_drag_x
```

### Plain meaning

- Soft bed / high water pressure → easier sliding (lower resistance).
- Sticky $C$ → harder sliding.
- The $\lambda$ GP still exists in the model object for SIA / older checkpoints, but **SSA physics no longer uses it**.

## Change 5 — Continuity residual optional

### Why

Original VI forced

$$
\nabla\cdot(H\mathbf{u})\approx 0
$$

as a third residual.

icepack’s **diagnostic** SSA velocity solve does **not** enforce that; thickness evolution is a *prognostic* step.

Enforcing continuity on a steady diagnostic snapshot can fight the same data icepack produced.

### Before

```python
rh = H_x * u + H * u_x + H_y * v + H * v_y
# always included in the physics log-likelihood
```

### After

```python
# CHANGED: optional; default False
if enforce_continuity:
    rh = H_x * u + H * u_x + H_y * v + H * v_y
else:
    rh = None
# only add rh term if not None
```

Config:

```ini
ssa_enforce_continuity = False
```

## Change 6 — Load $A$ and $C$ from spin-up `cfg_json`

### Why

Each spin-up case has its own $A$ (fluidity) and $C$ (friction), e.g. more_sliding has $C=0.001$, $A=20$. Hard-coding defaults would run different physics than the NPZ.

### Before

```python
# no cfg_json handling; friction_C / fluidity_A not used in SSA
```

### After

```python
def apply_spinup_cfg_to_pars(pars, cfg):
    """CHANGED: copy spin-up icepack parameters into the VI prior section."""
    if 'C' in cfg:
        pars.prior.friction_C = float(cfg['C'])
    if 'A' in cfg:
        pars.prior.fluidity_A = float(cfg['A'])

# inside load_snapshot:
if 'cfg_json' in data:
    apply_spinup_cfg_to_pars(pars, json.loads(str(data['cfg_json'])))
```

## What you should remember

1. We did **not** put icepack inside the PyTorch loop; we made VI’s SSA residual **look like icepack / spin-up**.
2. Biggest conceptual fix: $\lambda$ driving-stress sliding → plastic basal law with known $C$.
3. Units matter as much as equations.
4. Continuity is **optional** for diagnostic comparisons.
5. Search `# CHANGED` in `Archive/` to audit every physics edit.
